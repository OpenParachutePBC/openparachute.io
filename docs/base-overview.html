Server Lifecycle

server.py - Lifespan Manager
@asynccontextmanager
async def lifespan(app: FastAPI):
    """Application lifespan manager."""
    settings = get_settings()

    # Initialization
    setup_logging(level=settings.log_level)
    settings.vault_path.mkdir(parents=True, exist_ok=True)
    (settings.vault_path / ".parachute").mkdir(exist_ok=True)

    # Core services
    server_config = init_server_config(settings.vault_path)
    db = await init_database(settings.database_path)
    orchestrator = Orchestrator(
        vault_path=settings.vault_path,
        database=db,
        settings=settings,
    )

    # Background services
    curator = await init_curator_service(db, settings.vault_path)
    scheduler = await init_scheduler(settings.vault_path)

    # Store in app.state
    app.state.orchestrator = orchestrator
    app.state.database = db
    app.state.curator = curator
    app.state.scheduler = scheduler

    yield  # Server runs here

    # Shutdown
    await stop_scheduler()
    await stop_curator_service()
    await close_database()

Authentication

API key authentication with multiple modes:

Mode Behavior Use Case
disabled No auth required Local-only development
remote (default) Localhost bypasses, remote requires key Normal development
always All requests require valid API key Production / shared server
API Key Format
# Generated keys follow this format:
para_aBcDeFgHiJkLmNoPqRsT1234  # Full key (shown once)
k_aBcDeFgHiJkL                 # Key ID (for reference)

# Keys are SHA-256 hashed before storage:
sha256:abc123def456...         # Stored hash

Keys can be managed via:

  • POST /api/auth/keys - Create new key
  • GET /api/auth/keys - List keys (hashed)
  • DELETE /api/auth/keys/{key_id} - Revoke key
